# 开发测试

更新日期：2026-06-05

本文记录 Lapras 桌面版 Day7-Day11 开发测试中已经踩过的问题、根因和修复方案，主要用于后续打包、安装、首次引导、网易云接入和播放链路回归。

## 总体原则

- 打包后不要把 `resources/app.asar` 当成可写目录或子进程工作目录。
- 用户数据、数据库、画像缓存、日志必须写到 Electron `userData` 下的 `data` 目录。
- 本地后端 API 可以用 `x-lapras-token` 保护，但 `<audio>` 标签发起的音频请求不能附带自定义请求头，需要单独设计访问方式。
- 设置页和首次引导不要因为外部服务启动失败而整页卡死，非关键网络状态读取应使用降级或 `Promise.allSettled`。
- 每次修复打包问题后，必须用 `release/win-unpacked/Lapras.exe` 做一次打包态验证，源码态验证不够。

## 已解决问题

### 1. 安装协议中文乱码

现象：
Windows 安装器协议页中文显示乱码。

根因：
NSIS/安装协议文本编码与中文内容不兼容，打包后页面无法稳定显示中文。

修复方案：
将安装协议内容改为英文，避免编码兼容问题。

回归验证：
重新运行安装包，协议页应显示英文文本，无乱码。

### 2. 卸载删除数据后仍不触发首次引导

现象：
卸载时选择删除数据，重新安装后仍然没有进入首次引导。

根因：
首次引导状态、用户数据或配置清理不完整，导致新安装仍读取到旧的完成状态。

修复方案：
统一梳理首次引导状态触发条件，确保删除数据时清理 Electron `userData` 下的应用数据，重新安装后可重新进入 onboarding。

回归验证：
卸载时选择删除数据，重新安装并启动，应该进入首次引导；不删除数据时，应该保留已完成状态。

### 3. 跳过网易云登录后设置页黑屏或一直加载

现象：
首次引导跳过网易云登录后，进入主界面再打开设置页，只显示“正在加载设置”，页面无法正常渲染。

根因：
设置页初始化时把网易云状态、画像摘要等外部服务读取作为强依赖；网易云本地服务未启动或不可达时，整个设置加载流程被阻塞。

修复方案：
- 设置页核心数据使用 `Promise.all` 加载。
- 网易云状态、画像摘要等非关键数据改为 `Promise.allSettled`。
- `getNeteaseStatus()` 不主动启动网易云服务，只读取当前状态。

回归验证：
跳过网易云登录后，设置页应正常打开；网易云状态可以显示待登录或不可用，但不能阻塞整页。

### 4. 网易云二维码无法生成

现象：
首次引导或设置页点击“生成二维码”失败，提示无法连接 `http://127.0.0.1:3000`。

根因：
打包后不能依赖 `npx` 或外部命令启动网易云 API 服务；本地服务没有稳定随 Lapras 启动。

修复方案：
- 将 `@neteasecloudmusicapienhanced/api` 加入正式依赖。
- 增加 `build/netease-service.cjs` 作为服务 wrapper。
- 打包时把 wrapper 放入 `app.asar`。
- Electron 主进程提供 `lapras:local-services:ensure` IPC，由前端在二维码、验证码、Cookie 导入等动作前确保服务已启动。
- 网易云本地服务端口从 `3000` 开始探测，支持 `3000-3009` 自动避让。

回归验证：
打包后首次引导点击生成二维码，应自动启动本地网易云服务并显示二维码。

### 5. 点击生成二维码没有反应

现象：
设置页可以打开，但点击“生成二维码”后界面看起来没有变化。

根因：
网易云服务子进程启动时 `cwd` 被设置为 `resources/app.asar`。该路径不是普通目录，Windows 下会导致 `spawn ... ENOENT`，后端返回失败。

修复方案：
将网易云服务子进程工作目录改为真实安装目录：

```ts
cwd: getChildProcessCwd()
```

同时在设置页增加按钮级状态提示：“正在启动本地网易云服务并生成二维码...”，避免用户看到静默状态。

回归验证：
打包后用 `release/win-unpacked/Lapras.exe` 启动 wrapper，访问 `/inner/version` 应返回成功；请求二维码 key 和二维码图片也应成功。

### 6. 扫码成功后手动同步网易云画像失败

现象：
网易云账号显示在线，但点击“同步资料”无明显反应，底部提示：

```text
ENOTDIR, not a directory
```

根因：
`NeteaseAccountService.writeProfile()` 仍将 `netease_profile.json` 写入 `workspaceRoot/user`。打包后 `workspaceRoot` 指向 `resources/app.asar`，这是不可写路径。

修复方案：
- `ProfileService` 和 `NeteaseAccountService` 改用 `config.profileDataDir`。
- 打包后画像缓存写入 `%APPDATA%/Lapras/data/profiles`。
- “同步资料”按钮增加即时提示：“正在同步网易云画像...”。

回归验证：
扫码登录后进入设置页，点击同步资料，应能写入画像文件并刷新画像摘要；日志不应再出现 `ENOTDIR`。

### 7. 网易云登录后歌曲无法播放

现象：
点击播放失败，界面提示“播放失败，请尝试下一首”。日志中 `/api/audio/netease/...` 返回 `401`。

根因：
本地后端开启 `x-lapras-token` 鉴权后，普通 API 请求可以由前端附带 token，但 `<audio>` 标签加载音频时不能附带自定义 header，于是音频代理请求被本地鉴权拦截。

修复方案：
在本地后端鉴权 hook 中放行音频代理路由：

```ts
request.url.startsWith("/api/audio/")
```

这样设置页等普通 API 仍受 token 保护，而音频代理可以被媒体元素访问。

回归验证：
带 `LAPRAS_LOCAL_TOKEN` 启动后端，不带 token 请求 `/api/settings` 应返回 `401`；不带 token 请求 `/api/audio/netease/...` 不应返回 `401`，应进入音频代理逻辑。

### 8. Day7 沉浸感调整

现象：
切歌通知和任务栏小播放状态会打断用户沉浸感。

调整方案：
- 切歌不触发系统通知。
- 任务栏图标不展示小播放状态。
- Windows 任务栏悬停缩略图按钮保留上一首、播放/暂停、下一首。

回归验证：
切歌时没有系统通知；任务栏缩略图仍有媒体控制按钮。

## 打包态回归清单

每次涉及 Electron 主进程、安装器、网易云服务、数据目录、音频播放时，至少执行以下检查：

1. `npm.cmd run typecheck`
2. `npm.cmd run package:win`
3. 安装前从托盘彻底退出旧版 Lapras。
4. 运行新安装包，确认协议页无乱码。
5. 删除数据卸载后重装，确认首次引导会出现。
6. 首次引导生成网易云二维码，确认二维码可显示。
7. 扫码确认后，确认账号状态显示在线。
8. 打开设置页，确认不会卡在加载态。
9. 点击同步网易云画像，确认无 `ENOTDIR`，画像摘要刷新。
10. 播放一首网易云歌曲，确认 `/api/audio/netease/...` 不再返回 `401`。
11. 查看 `%APPDATA%/Lapras/data/logs`，确认没有新的 P0 错误。

## 关键路径约定

开发态：

```text
D:/Lapras/lapras-desktop/data
D:/Lapras/lapras-desktop/user
```

打包态：

```text
%APPDATA%/Lapras/data
%APPDATA%/Lapras/data/profiles
%APPDATA%/Lapras/data/logs
```

禁止在打包态写入：

```text
resources/app.asar
resources/app.asar/user
resources/app.asar/data
```

## 后续注意事项

- 新增任何文件缓存时，优先接入 `config.dataDir` 或 `config.profileDataDir`。
- 新增任何子进程时，打包态 `cwd` 必须使用真实目录，例如 `path.dirname(process.execPath)`。
- 新增媒体资源代理时，先确认请求是否由浏览器元素直接发起；如果不能带 header，就不要套用普通 API 鉴权方式。
- 新增设置页数据源时，区分“页面必需数据”和“外部服务状态数据”，避免一个外部服务失败拖垮整页。
- 每次安装器变更后，都要做卸载、删除数据、重装三连测。
